D2. FAB DIRECT FORM OPENING
---------------------------
New utility: frontend/src/utils/portalFormIntent.js
  - queuePortalFormIntent(tabId, 'add')
  - consumePortalFormIntent(tabId) — one-shot on tab mount

New hook: frontend/src/hooks/usePortalFormIntent.js
  - useEffect on tab mount consumes intent and runs handler

App.jsx navigateTab extended:
  navigateTab(tabId, { openForm: 'add' }) → queues intent then sets activeTab

  onNavigate(action.tab, { openForm: 'add' })

Limitation: Does NOT pre-fill or deep-link to a specific record from search.
Search still navigates to tab only.

D8. ANDROID BACK BUTTON (USER BUG #3)
--------------------------------------
New: frontend/src/hooks/useMobileBackNavigation.js

Lifted overlay state to App.jsx (was inside MobilePortalShell):
  searchOpen, reportsOpen, fabOpen (+ existing notificationsOpen, sidebarOpen)

Behavior on popstate (mobile only):
  1. Close overlay priority: notifications → search → reports → FAB → sidebar
  2. Else pop tab stack → navigateTab(previous, { fromBack: true })
  3. At root tab: window.confirm("Leave VouchEx?...")
     - OK → history.back() (exit)
     - Cancel → pushState guard (stay)

Uses history.pushState on tab change to build stack.

Limitation:
  - Does not handle in-tab form back (e.g. close add form on back)
  - Does not handle Debtors party drill-down back
  - confirm() is browser native dialog (not branded sheet)

PART H — KNOWN GAPS / WHY NOT 10/10 YET (FOR CHATGPT)
================================================================================

1. REGISTRY INCONSISTENCY (biggest remaining issue)
   Credit notes, debit notes, registers, COA, cash-bank still horizontal tables.
   User psychologically hits "desktop page" on these screens.

2. SEARCH NOT TRUE DEEP LINK
   Finds record but only opens parent tab — cannot scroll/highlight/open that row.

3. FAB / BACK NOT FORM-AWARE
   Hardware back does not close an open add form inside a tab.
   User may expect back to exit invoice form before exiting tab.

4. RECEIPT/PAYMENT SUB-TABLES
   Pending invoices / pending bills still full-width tables inside cardified pages.

5. GSTR MOBILE SUMMARY ONLY ON GSTR-1
   GSTR-2B/3B still dump full tables immediately — no summary-first pattern.

6. OVERFLOW FIX IS GLOBAL CLIP
   overflow-x: clip prevents page bleed but some edge pages may still feel tight;
   not all inline grid layouts converted to responsive classes.

7. DEBTORS OVERDUE HEURISTIC
   Uses >30 days since last invoice — not true ageing bucket from API on cards.

8. NO SWIPE GESTURES / NO NATIVE APP FEEL
   Still browser SPA; no pull-to-refresh, no haptic, no shared element transitions.

9. COMMAND PALETTE INCOMPLETE
   Quick jump is tab-level only — no "Record receipt", "Raise invoice" shortcuts
   in palette (FAB handles create but not searchable by voice-of-command).

10. PARTY DRILL-DOWN STILL TABLE-HEAVY
    Debtors/Creditors "View details" opens nested tables not cards.
