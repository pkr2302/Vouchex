SECTION 1 — INTRODUCTION TO VOUCHEX
================================================================================

VouchEx is a cloud-based GST accounting and business-management portal built for
Small & Medium Enterprises (SMEs) and Chartered Accountants (CAs) in India.

Tagline: "Accounts Made Simple"

It is NOT a native iOS/Android app. It is a responsive single-page web application
(SPA) built with React 19 + Vite on the frontend and Laravel 11 API on the backend.
Users access it through mobile browsers (Chrome, Safari, etc.) on phones and tablets.

Architecture:
- Multi-company SaaS: one login can manage multiple isolated companies (super admin)
- Single database with company_id isolation per tenant
- Desktop-first UI inspired by ClearBooks-style accounting portals
- Mobile support added via CSS media queries (not a separate mobile app)

Primary audience:
- Business owners who need invoicing, expenses, GST compliance on the go
- CAs managing multiple client companies
- Trial users evaluating the product from marketing landing page


SECTION 2 — WHAT VOUCHEX OFFERS (VALUE PROPOSITION)
================================================================================

Core promise:
Complete day-to-day accounting + GST compliance in one browser portal, without
installing desktop software.

What users can do end-to-end:
1. Register / trial / subscribe (SaaS onboarding)
2. Set up company profile (GSTIN, PAN, bank, accounting framework AS or Ind AS)
3. Maintain masters: customers, vendors, products/services, chart of accounts
4. Record commercial documents: sales invoices, purchase bills, expenses, receipts,
   payments, credit notes, debit notes
5. Manage inventory, stock consumption, cash & bank ledgers
6. View debtors/creditors ageing, party ledgers, forex receipts
7. Generate financial statements (Trial Balance, P&L, Balance Sheet, Notes)
8. GL registers: ledgers, day book, sales register, purchase register, misc entries
9. GST taxation workspace: GSTR-1/2B/3B views, TDS, compliance calendar, exports
10. Settings: users, financial year lock, backups, audit trail, subscriptions
11. E-invoice / e-way bill readiness (where enabled)
12. OCR bill scanning simulation on invoices/expenses
13. PDF generation and export (Excel/CSV) for reports and tax filings

Accounting depth:
- Double-entry general ledger (GL) with automatic posting from vouchers
- Supports AS and Ind AS accounting frameworks
- Report groups, COA auto-mapping, guided journal entries
- Financial period filtering (FY, quarter, half-year, custom dates)

Compliance focus:
- Indian GST (CGST/SGST/IGST bifurcation, place of supply, HSN/SAC)
- TDS receivable/payable tracking
- Statutory calendar with tasks and reminders
- GSTR-oriented register exports

GLOBAL PORTAL SHELL (all logged-in pages)
-----------------------------------------

Desktop (width > 1024px):
  Layout: flex row
  - Left sidebar: fixed width 280px (--sidebar-width), sticky, full viewport height
    Dark navy gradient, logo top, grouped menu scrollable, company badge footer
  - Main content: flex-grow, min-height 100vh
    - Sticky header: 70px height, page title left, user actions right
    - Tab container: padded content area (32px desktop padding)

Mobile / tablet (width <= 1024px):
  Layout: block (single column)
  - Sidebar: position FIXED, off-screen left (translateX -100%), width min(280px, 88vw, 300px)
    Slides in when open; z-index 1000; shadow
  - Sidebar close button (X): top-right inside sidebar logo row, 40×40px
  - Backdrop: fixed full-screen rgba overlay z-index 999 when menu open
  - Scroll lock: html gets class nav-scroll-lock when menu open (body overflow hidden)
  - Main content: width 100%, max-width 100vw, overflow-x hidden, vertical scroll
  - Hamburger button: 40×40px, grid area "menu" in header, toggles sidebar
  - Header: CSS grid 3-row layout on mobile:
      Row 1: [hamburger] [page title + "Portal" label + clock] [actions]

SECTION 7 — MOBILE CSS BREAKPOINTS & RESPONSIVE RULES
================================================================================

Primary breakpoints used across App.css:
  - 1200px: some dashboard/chart grids
  - 1100px: dashboard layout columns, compliance grids
  - 1024px: MAIN mobile portal breakpoint — sidebar becomes drawer, hamburger visible
  - 960px: assorted form/layout tweaks
  - 768px: forms to 1 column, sub-tab horizontal scroll, financial period bar stack,
           header clock hidden, auth/subscription adjustments
  - 640px: phone polish — safe areas, 2-col quick actions, single-col attention cards,
           modals full width, company select shrink, user name hidden

Touch targets (mobile intent):
  - Many buttons/inputs bumped to min-height 44px on financial/register areas
  - Date/select inputs font-size 16px on mobile (prevents iOS zoom-on-focus)

Tables:
  - Wrapped in .premium-table-wrapper with overflow-x auto and -webkit-overflow-scrolling touch
  - Most data-heavy screens expect HORIZONTAL SCROLL on phone rather than card transformation
  - Exceptions: manual journal table uses card-stack pattern; GST register hides columns

Forms:
  - .form-grid-2, .form-grid-3, .form-grid-4 → single column at <=768px

Sub-tab navigation (.tab-nav-sub):
  - flex-wrap nowrap + overflow-x auto + hidden scrollbar on mobile
  - User swipes horizontally to reach later tabs (Payments has 4, Taxation has 6, Settings many)

Viewport meta (index.html):
  width=device-width, initial-scale=1.0, viewport-fit=cover


SECTION 8 — MAIN MOTTO & MOBILE STRATEGY (WHAT WE ARE OPTIMIZING FOR)
================================================================================

Motto:
  "Full accounting portal accessible anywhere — not a cut-down companion app."

Mobile strategy (current):
  - Same feature set as desktop (no feature gating by screen size)
  - Navigation compressed into hamburger drawer instead of persistent sidebar
  - Content reflows to single column; wide tables scroll horizontally
  - Critical workflows (quick actions, attention alerts, create invoice/expense) reachable
    from dashboard within 1–2 taps
  - Touch-friendly targets on financial/register flows
  - Safe-area support for modern phones

NOT current goals:
  - Native app gestures
  - Offline mode
  - Bottom tab bar navigation
  - Separate simplified "mobile-only" invoice creator
  - Progressive Web App install prompt (not implemented)

Primary mobile user stories we care about:
  1. Check dashboard alerts and compliance status on phone
  2. Quickly add receipt, expense, or view overdue debtors
  3. Look up a customer/vendor or invoice while on call
  4. Review P&L or GST summary before filing
  5. Super admin switches company and checks multiple clients

Known weak areas (honest gaps for reviewer):
  1. Invoice/expense LINE ITEM tables — wide multi-column; horizontal scroll only
  2. Taxation GSTR tables — very wide; mobile review is hard
  3. Company 360 charts — desktop-oriented draggable grid
  4. Settings with many sub-tabs — requires horizontal swipe to find tab
  5. Chart of Accounts report groups tree — complex on narrow screens
  6. Console drawer / developer log — may overlap mobile viewport bottom
  7. Some action button clusters in table rows wrap but can feel cramped
  8. Recent fix (June 2026): sidebar CSS cascade bug caused content shifted off-screen
     on mobile — fixed by enforcing position:fixed sidebar after base styles; needs
     production deploy verification

Desktop unchanged above 1025px width — all mobile rules inside max-width media queries.
